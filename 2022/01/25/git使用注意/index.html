git使用注意

发表于 2022-01-25 更新于 2022-07-18

我啥时候搞了这个，算了为了避免时常抽风忘记所以也不错。

转换上游地址

git remote add 名称 url

要换先 remove 再 add

pull 多个的时候指定分支， 

git pull origin main

哎呀呀呀，慢慢写笔记

就首先那个 gitignore，py 要不记录缓存可以用

pycache/
*.py[cod]

在 github 似乎有专门的一个 repo 示范

分支会积累很多，记得定期删除不必要的分支，只拉取需要的分支。转到那个分支上再 git pull 这样就只拉取了一个，一般是去 main

git branch -d <分支名>
